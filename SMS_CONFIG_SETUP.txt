# SMS API 配置说明
# 请将以下信息替换为您真实的SMS API配置

# SMS Console 配置示例:
# API URL: https://www.sms-console.jp/api/
# API ID: 您的SMS Console用户ID
# API Password: 您的SMS Console密码

# 修改方法：
# 1. 打开 src/backend/rpa_server.js 文件
# 2. 找到第327-330行左右的配置部分
# 3. 将 YOUR_API_ID_HERE 替换为您的真实API ID
# 4. 将 YOUR_API_PASSWORD_HERE 替换为您的真实API密码
# 5. 保存文件并重启服务器

# 或者您也可以：
# 1. 在网页界面的"各種設定"中设置SMS配置
# 2. 然后切换回原来的 send_personal_sms.py 脚本（需要Firebase服务账户密钥）

# 当前使用的是临时解决方案，绕过了Firebase配置获取
